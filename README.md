# Mnemonic Map Forge

Mnemonic Map Forge 是一个用于创建“记忆漫游图 / memory exploration maps”的技能项目。它专注于让知识图谱更适合记忆而不是严格分类，通过低噪声连接、可复用模板和辅助节点来强化联想记忆。

## 项目目标

- 生成**以记忆优先**的概念图，而不是标准的知识图谱
- 保持图形连通、可走、低边密度
- 使用**辅助节点**压缩记忆负担
- 避免噪声边、双向重复边和过度分类
- 通过模板化输出支持可复用的 PyVis 渲染

## 适用场景

当你需要：

- 将学习内容转成记忆友好的图形
- 把笔记、框架、方法、流程或对比信息变成可漫游的记忆地图
- 让用户通过少量关键路径快速回忆起大量概念
- 优先“联想记忆”而非“严格本体建模”

## 核心原则

1. 只保留能帮助快速记起的边
2. 一张图必须是单连通的
3. 边密度必须受控，通常低于 1.2 条/节点
4. 辅助节点只在能压缩记忆时才使用
5. 不要同时创建 `A → B` 和 `B → A` 的重复关系

## 输出结构概览

Mnemonic Map Forge 的目标输出通常包含：

- `clusters`：节点分区、颜色和布局信息
- `nodes`：真实记忆节点，带标签、提示、tooltip 等
- `helpers`：辅助节点，用于压缩记忆连接
- `edges`：记忆路径边，支持方向性和语义关联

示例数据结构：

```json
{
  "clusters": {
    "A. Example": {
      "color": "#e74c3c",
      "center": [0, 0],
      "nodes": [
        {"id": "node_a", "label": "A", "title": "A", "tooltip": "A\n核心记忆", "nodeType": "tool"}
      ]
    }
  },
  "helpers": [
    {"id": "helper_x", "label": "◈ X", "tooltip": "helper", "connects_to": ["node_a"], "position": [180, -40]}
  ],
  "edges": []
}
```

## 贡献建议

如果你希望改进本项目：

- 优先检视 `SKILL.md` 中的记忆图规范
- 保持边密度低、图结构清晰
- 通过实际案例验证图是否更易于记忆
- 避免把本项目变成“纯分类图”或“超密知识图谱”

## 备注

这个项目更像是一个“图形设计/记忆策略”技能，而不是一个通用渲染引擎。真正的价值在于用对连接、辅助节点和布局策略，帮助用户把复杂内容编织成可回忆的记忆地图。
